Qué es OpenVidu?

OpenVidu es una plataforma de videollamada enfocada especialmente para aplicaciones web o aplicaciones moviles. OpenVidu funciona como una API REST, la cuál, tiene licencia Apache license v2 y para su ejecución se requiere tener instalado Java 8.

Qué características tiene OpenVidu?

  • Llamadas de uno a uno, de uno a muchos y de muchos a muchos.
  • Decido qué usuario puede publicar y cuál no con roles predefinidos.
  • Enviar transmisiones de audio + video, solo audio o solo video.
  • Transmitir mensajes de texto.
  • Silencia cualquier pista cuando quieras.
  • Compartir tu pantalla.
  • Graba tus videollamadas.

Qué se necesita para el funcionamiento de la API REST?

Para el funcionamiento de OpenVidu es recomendable crear un VPS eligiendo el Sistema Operativo Ubuntu xenial 16.04 y luego, ejecutar los siguientes parametros dentro de la VPS creada:

1. Instalar KMS

sudo echo "deb http://ubuntu.openvidu.io/6.7.2 xenial kms6" | sudo tee /etc/apt/sources.list.d/kurento.list
sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys 5AFA7A83
sudo apt-get update
sudo apt-get -y install kurento-media-server

2. Instalar COTURN

sudo apt-get -y install coturn

3. Instalar Redis

sudo apt-get -y install redis-server

4. Modificar en el terminal la siguiente información:

/etc/kurento/modules/kurento/WebRtcEndpoint.conf.ini

Y colocar el siguiente codigo:

stunServerAddress=YOUR_MACHINE_PUBLIC_IP
stunServerPort=3478

5. Modificar en el terminal la siguiente información:

/etc/turnserver.conf

Y colocar el siguiente codigo:

external-ip=YOUR_MACHINE_PUBLIC_IP
listening-port=3478
fingerprint
lt-cred-mech
max-port=65535
min-port=49152
pidfile="/var/run/turnserver.pid"
realm=openvidu
simple-log
redis-userdb="ip=127.0.0.1 dbname=0 password=turn connect_timeout=30"
verbose

6. Modificar en el terminal la siguiente información:

/etc/default/coturn

Y colocar el siguiente codigo:

TURNSERVER_ENABLED=1

7. Inicio de Servicios

sudo service redis-server restart
sudo service coturn restart
sudo service kurento-media-server restart

8. Iniciar el ejecutable de OpenVidu Server JAR

java -jar -Dopenvidu.secret=YOUR_SECRET -Dopenvidu.publicurl=https://YOUR_MACHINE_PUBLIC_IP:4443/ openvidu-server.jar &
  • Necesitará Java 8 para ejecutar el Servidor OpenVidu:

sudo apt-get install -y openjdk-8-jre

  • Puede ejecutar cualquier versión de OpenVidu Server:

wget https://github.com/OpenVidu/openvidu/releases/download/v{VERSION}/openvidu-server-{VERSION}.jar - Se recomienda la versión 2.4.0 de OpenVidu Server

API REST bajo Ruby on Rails

Para simplificar los llamados a esta API REST, se creó una clase bajo lib/ov.rb y un método request que puede llamar hacer post a los dos endpoints mas importantes, el de sessions y el de token.

Estos endpoints permiten crear una sesión (esto en humano es iniciar la transmisión del video) y obtener un token (el cual sirve para conectarse a una transmisión existente)

Para iniciar una transmisión

Se debe llamar al endpoint de session

Ov.request("api/sessions", "post", {customSessionId: @course.name})

El resultado es un token que se necesita en el js para unirse a la sesión recién creada

Ov.request("api/tokens", "post", {session: @course.name})

El resultado es un token que se necesita en el js para unirse a la sesión existente

Actualmente dentro de la vista courses#show está el javascript que usa el token para crear o unirse a la sesión.

Reglas importantes:

  • No se puede unir a una transmisión sin un token
  • No se puede unir a una sesión que no existe
  • No se pueden crear dos sesiones con el mismo nombre.
